## Overview
할 일을 칸반 보드로 시각적으로 관리하는 앱. 카드를 컬럼 간 드래그&드롭으로 이동하고, 우선순위·태그·서브태스크로 세부 관리한다.

## 시나리오

### 1. 카드 추가
[상황] 사용자가 칸반 보드의 특정 컬럼 하단 "카드 추가" 버튼을 클릭하면
[동작] 시스템이 해당 컬럼에 제목 입력 폼을 표시하고, 입력 후 추가하면 카드가 해당 컬럼에 나타난다

성공 기준:
- [ ] 제목 "회의록 작성" 입력 후 추가 → "할 일" 컬럼에 "회의록 작성" 카드 표시
- [ ] 제목 빈 채로 추가 시도 → "제목을 입력해주세요" 오류 메시지 표시, 카드 추가 안 됨

### 2. 카드 인라인 편집
[상황] 칸반 보드에 카드가 있을 때
[동작] 사용자가 카드 제목을 클릭하면 인라인으로 수정 가능하고, 더보기 패널에서 나머지 필드(설명·우선순위·태그·서브태스크)를 편집한다

성공 기준:
- [ ] 제목 "회의록 작성" 클릭 → 제목이 편집 가능한 입력란으로 변환됨
- [ ] 제목을 "기획 미팅 정리"로 수정 후 포커스 해제 → 카드에 "기획 미팅 정리" 표시
- [ ] 더보기 패널 열기 → 설명, 우선순위(High/Medium/Low 선택), 태그 입력란, 서브태스크 목록 표시
- [ ] 우선순위 "High" 선택 → 카드에 "High" 배지 표시
- [ ] 태그 "#업무" 입력 후 추가 → 카드에 "#업무" 태그 표시

### 3. 카드 삭제
[상황] 칸반 보드에 카드가 있을 때
[동작] 사용자가 카드의 삭제 버튼을 클릭하면 해당 카드가 보드에서 제거된다

성공 기준:
- [ ] "회의록 작성" 카드의 삭제 버튼 클릭 → 카드가 목록에서 사라짐
- [ ] 삭제 후 해당 컬럼 카드 수 1 감소

### 4. 드래그&드롭으로 카드 이동
[상황] 칸반 보드에 카드가 있을 때
[동작] 사용자가 카드를 다른 컬럼 또는 같은 컬럼 내 다른 위치로 드래그하면 카드가 이동된다

성공 기준:
- [ ] "할 일" 컬럼의 카드를 "진행 중" 컬럼으로 드래그 → "진행 중" 컬럼에 카드 표시, "할 일" 컬럼에서 제거
- [ ] 같은 컬럼 내 카드 순서 드래그 변경 → 변경된 순서로 카드 표시
- [ ] 이동 후 페이지 새로고침 → 이동된 위치 유지

### 5. 서브태스크 체크리스트
[상황] 카드의 더보기 패널이 열려 있을 때
[동작] 사용자가 서브태스크를 추가·체크·삭제하면 체크리스트와 진행률이 업데이트된다

성공 기준:
- [ ] 서브태스크 "자료 조사" 추가 → 체크리스트에 "자료 조사" 항목 표시
- [ ] 서브태스크 3개 중 1개 체크 → 카드에 "1/3" 진행률 표시
- [ ] 서브태스크 3개 중 3개 체크 → 카드에 "3/3" 진행률 표시
- [ ] 서브태스크 항목 삭제 → 해당 항목이 목록에서 제거됨

### 6. 제목 검색
[상황] 칸반 보드에 여러 카드가 있을 때
[동작] 사용자가 검색창에 키워드를 입력하면 제목에 해당 키워드를 포함한 카드만 표시된다

성공 기준:
- [ ] 검색어 "회의" 입력 → "회의록 작성", "팀 회의 준비" 카드만 표시
- [ ] 검색어 "xyz" 입력 → 모든 컬럼에 카드 0개 표시, "검색 결과가 없습니다" 안내 문구
- [ ] 검색어 지우기 → 전체 카드 다시 표시

### 7. 우선순위·태그 필터
[상황] 칸반 보드에 여러 카드가 있을 때
[동작] 사용자가 우선순위 또는 태그 필터를 선택하면 해당 조건에 맞는 카드만 표시된다

성공 기준:
- [ ] 우선순위 "High" 필터 선택 → High 우선순위 카드만 표시
- [ ] 태그 "#업무" 필터 선택 → "#업무" 태그가 있는 카드만 표시
- [ ] 우선순위 "High" + 태그 "#업무" 동시 선택 → 두 조건 모두 만족하는 카드만 표시
- [ ] 필터 초기화 → 전체 카드 다시 표시

### 8. 컬럼 이름 변경
[상황] 칸반 보드가 표시된 상태에서
[동작] 사용자가 컬럼 이름을 더블클릭하면 인라인으로 이름을 수정할 수 있다

성공 기준:
- [ ] "할 일" 컬럼 이름 더블클릭 → 이름이 편집 가능한 입력란으로 변환
- [ ] "할 일"을 "백로그"로 수정 후 포커스 해제 → 컬럼 헤더에 "백로그" 표시
- [ ] 이름을 빈 값으로 저장 시도 → 기존 이름 유지

### 9. 다크모드 토글
[상황] 앱이 라이트모드로 표시된 상태에서
[동작] 사용자가 다크모드 토글을 클릭하면 전체 UI가 다크모드로 전환된다

성공 기준:
- [ ] 다크모드 토글 클릭 → 배경이 어두운 색상으로 변환, 토글이 "on" 상태로 표시
- [ ] 다크모드 상태에서 새로고침 → 다크모드 유지

### 10. localStorage 저장·복원
[상황] 카드 추가/이동/편집/삭제, 컬럼 이름 변경 후
[동작] 페이지를 새로고침해도 데이터가 유지된다

성공 기준:
- [ ] "회의록 작성" 카드 추가 후 새로고침 → "회의록 작성" 카드 표시
- [ ] 카드를 "완료" 컬럼으로 이동 후 새로고침 → "완료" 컬럼에 카드 표시
- [ ] 컬럼 이름을 "백로그"로 변경 후 새로고침 → "백로그"로 표시

## 범위

### 포함
- 3컬럼 칸반 보드 (이름 변경 가능, 개수 고정)
- 카드 CRUD (제목 필수, 설명/우선순위/태그/서브태스크 선택)
- 카드 인라인 편집 (제목 직접 클릭, 나머지는 더보기 패널)
- 드래그&드롭 (컬럼 간 이동, 같은 컬럼 내 순서 변경)
- 서브태스크 체크리스트 + 진행률 표시
- 제목 실시간 검색
- 우선순위·태그 필터 (복합 조건)
- 다크모드 토글 + localStorage 저장
- 전체 데이터 localStorage 저장·복원

### 제외
- 마감일(due date) 필드: 범위 집중을 위해 제외 (추후 확장 가능)
- 컬럼 추가·삭제: 데이터 모델 복잡도 증가 대비 이익 낮음
- 백엔드/서버 동기화: localStorage 단독 저장으로 결정
- 카드 간 의존성 표시: 복잡도 높음
- 사용자 인증: 단일 사용자 앱

## 전제 조건
- 초기 컬럼 이름: "할 일", "진행 중", "완료"
- 우선순위: High / Medium / Low (미선택 허용)
- 태그: 카드별 자유 입력, 복수 태그 가능
- 검색·필터는 실시간(입력/선택 즉시 반영)
- 다크모드 상태는 localStorage에 저장

## 보강 시나리오 (검토 후 추가)

### 11. 더보기 패널 열기 - 편집 필드 표시
[상황] 칸반 보드에 카드 1개가 표시된 상태에서
[동작] 사용자가 카드의 더보기 버튼을 클릭하면 편집 패널이 열리며 모든 필드가 표시된다

성공 기준:
- [ ] 더보기 버튼 클릭 → 설명 입력란, 우선순위 선택, 태그 입력란, 서브태스크 목록 표시

### 12. 드래그&드롭 이동 위치 새로고침 후 유지
[상황] 카드를 드래그로 다른 컬럼으로 이동한 상태에서
[동작] 사용자가 페이지를 새로고침하면 이동된 위치가 유지된다

성공 기준:
- [ ] '완료' 컬럼으로 드래그 후 새로고침 → '완료' 컬럼에 카드 존재, '할 일' 컬럼에 없음

### 13. 필터 초기화 후 전체 카드 복원
[상황] 우선순위 필터가 적용된 상태에서
[동작] 사용자가 필터를 초기화하면 모든 카드가 다시 표시된다

성공 기준:
- [ ] 필터 초기화 → 전체 카드 3개 표시

### 14. 카드 제목 인라인 편집 - 빈 값 방지
[상황] 기존 카드가 인라인 편집 모드인 상태에서
[동작] 사용자가 제목을 비우고 포커스를 해제하면 기존 제목이 유지된다

성공 기준:
- [ ] 제목 입력란을 비우고 포커스 해제 → 기존 제목 유지

### 15. 다크모드에서 라이트모드로 복귀
[상황] 앱이 다크모드로 표시된 상태에서
[동작] 사용자가 토글을 다시 클릭하면 라이트모드로 전환된다

성공 기준:
- [ ] 다크모드 토글 재클릭 → 라이트모드 전환, 토글 비활성 상태
